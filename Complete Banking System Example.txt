✅ Complete Banking System Example
1. User Stories (15)

Below are 15 well-formed user stories following the standard format:

Epic 1 — Customer Account Management
US-01: User Registration & Identity Verification

As a new user, I want to register and verify my identity so that I can access banking services.

US-02: Login & Multi-Factor Authentication

As a user, I want to securely log in with MFA so that I can protect my account.

US-03: View Account Summary

As a customer, I want to view my account balances so I can monitor my financial status.

Epic 2 — Money Movement
US-04: Transfer Money Between My Accounts

As a customer, I want to move money between my own accounts so I can manage my funds.

US-05: Transfer Money to External Accounts

As a customer, I want to send money to accounts at other banks so I can pay others.

US-06: Schedule Recurring Transfers

As a customer, I want to schedule recurring transfers so bills or savings happen automatically.

Epic 3 — Payments & Cards
US-07: Pay Bills

As a customer, I want to pay utility bills from my account so I can manage my expenses.

US-08: Debit Card Management

As a customer, I want to freeze or unfreeze my debit card so I can prevent unauthorized use.

Epic 4 — Loans
US-09: Apply for a Loan

As a customer, I want to apply for a loan online so I can avoid visiting a branch.

US-10: View Loan Details & Payment Schedule

As a borrower, I want to see my loan balance and upcoming payments so I can plan my finances.

Epic 5 — Statements & Audit
US-11: Download Monthly Statements

As a customer, I want to download account statements so I can track my transactions.

US-12: Transaction Search & Filtering

As a customer, I want to search my transactions by date or keyword so I can find specific charges.

Epic 6 — Notifications & Alerts
US-13: Low Balance Alerts

As a customer, I want alerts for low balances so I can avoid overdrafts.

US-14: Transaction Alerts

As a customer, I want notifications for large or suspicious transactions to detect fraud.

Epic 7 — Administration / Compliance
US-15: Admin User Management

As an admin, I want to manage user accounts and lock/unlock them to ensure compliance and security.

3. Acceptance Criteria (Per User Story)
US-01: User Registration & Identity Verification (8 pts)

Acceptance Criteria:
✔ User can enter personal info: name, phone, email, SSN/ID.
✔ System validates email and phone via OTP.
✔ System checks identity via government ID / verification API.
✔ Account is created only after verification passes.
✔ Error shown for duplicate or invalid identity.

US-02: Login & Multi-Factor Authentication (5 pts)

Acceptance Criteria:
✔ User can log in with username/password.
✔ MFA (SMS/Email/Auth App) required.
✔ Failed attempts lock account after configurable threshold.
✔ User can request password reset securely.

US-03: View Account Summary (3 pts)

Acceptance Criteria:
✔ User sees all checking, savings, and loan accounts on dashboard.
✔ Balances and available funds displayed accurately.
✔ Data updates in real time.

US-04: Transfer Money Between My Accounts (5 pts)

Acceptance Criteria:
✔ User can choose source and destination accounts.
✔ Transfer completes immediately if sufficient funds.
✔ Updated balances shown after transfer.
✔ Error message displayed if insufficient funds.

US-05: Transfer Money to External Accounts (13 pts)

Acceptance Criteria:
✔ User can add payee with routing/account number.
✔ Payee must be verified before transfer.
✔ Transfer uses ACH or wire depending on selection.
✔ System shows estimated delivery time.
✔ Confirmation and receipt generated.

US-06: Schedule Recurring Transfers (8 pts)

Acceptance Criteria:
✔ User can choose daily/weekly/monthly schedules.
✔ Transfers trigger automatically at the scheduled time.
✔ User can pause or cancel recurring transfers.
✔ Notifications sent upon success or failure.

US-07: Bill Payments (8 pts)

Acceptance Criteria:
✔ User can add billers (utilities, credit cards, etc.).
✔ Payment amount and date can be selected.
✔ Confirmation of payment shown on screen and stored.
✔ System warns user if balance is insufficient.

US-08: Debit Card Freeze/Unfreeze (5 pts)

Acceptance Criteria:
✔ User can toggle card status between active and frozen.
✔ Changes take effect instantly.
✔ System blocks transactions during freeze.

US-09: Apply for a Loan (13 pts)

Acceptance Criteria:
✔ User fills loan application with income/employment data.
✔ System runs automated eligibility and credit checks.
✔ Approval/denial status shown to user.
✔ Approved loans create an active loan account.

US-10: View Loan Details & Schedule (5 pts)

Acceptance Criteria:
✔ User sees loan balance, interest rate, and amortization schedule.
✔ User can view next payment due date.
✔ User can download payment schedule.

US-11: Download Monthly Statements (3 pts)

Acceptance Criteria:
✔ User can pick statement month/year.
✔ Statements generated as PDF.
✔ Download completes within seconds.
✔ Statement includes official bank branding.

US-12: Transaction Search & Filter (5 pts)

Acceptance Criteria:
✔ User can filter by date range, amount, type, keyword.
✔ Search returns results within 2 seconds (optimized).
✔ All details included: merchant, timestamp, category.

US-13: Low Balance Alerts (3 pts)

Acceptance Criteria:
✔ User can set custom threshold levels.
✔ Alerts sent via SMS, email, or push.
✔ Alert triggers immediately after threshold is crossed.

US-14: Suspicious Transaction Alerts (8 pts)

Acceptance Criteria:
✔ System detects unusual spending patterns.
✔ Alerts sent instantly to user.
✔ User can report a transaction as fraudulent.
✔ Fraud-alerted card auto-freezes until resolved.

US-15: Admin User Management (8 pts)

Acceptance Criteria:
✔ Admin can search customers by name, email, or account.
✔ Admin can lock/unlock accounts.
✔ All admin actions logged for audit.
✔ Permissions restricted by admin role.
