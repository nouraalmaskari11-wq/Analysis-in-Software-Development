3. User Stories
3.1 Definition

A User Story is a short, simple description of a feature told from the perspective of the end-user.

Standard Format

As a (type of user)
I want (goal)
So that (value/benefit)

3.2 How to Write Effective User Stories

Use clear, simple language

Focus on user value, not technical tasks

Ensure each story is Independent, Negotiable, Valuable, Estimable, Small, Testable (INVEST)

Add acceptance criteria

3.3 Acceptance Criteria

Conditions that must be met for the story to be accepted.

Importance

Remove ambiguity

Define “Done”

Support test case creation

Ensure alignment between developers and stakeholders
